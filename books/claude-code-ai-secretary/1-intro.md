## この本の前提

Claude Codeを「使う」のではなく「育てる」。この本はその実験の記録です。

2026年4月末からClaude Codeを個人AI秘書として運用し始めました。最初は単なるCLIツールとして使っていましたが、2ヶ月後にはこうなっています。

- 会話するだけで勝手にメモリを更新する自律学習プロトコル
- CEO→research-loop→researcherの3階層エージェント構造
- リサーチを自律的にループで回し、カバレッジが十分になるまで繰り返すシステム
- 17件の失敗ログから生まれたガードレール群
- 意思決定の反証ループ（結論に対して自ら反証リサーチを走らせ、生き残った結論だけを採用する仕組み）

これはチュートリアルではありません。「こう設定すればうまくいく」という話ではなく、「こう設定して、こう壊れて、こう直した」という実録です。

## 誰に向けた本か

Claude Codeをすでに使っていて、以下のどれかに心当たりがある人に向けています。

- 毎回同じ説明をClaude Codeにしている（前回のセッションの記憶がない）
- リサーチを頼むと浅い検索結果の寄せ集めが返ってくる
- 成果物の品質にムラがある
- エージェントを作ったが「使うと便利」止まりで、勝手に成長してくれない

## この本で扱うこと

1. **メモリ設計** — セッションをまたいで記憶が蓄積し、自律的に更新される仕組み
2. **3階層エージェント構造** — 戦略層（CEO）・実行層（research-loop）・収集層（researcher）の分離と、なぜ2階層ではダメだったか
3. **リサーチループ** — 「何を調べるべきか」を調べてから調べる、メタリサーチ先行のプロトコル
4. **17件の失敗** — 実運用で壊れたこと全部。確証バイアス、コンテキスト混同、独断修正、ベンダー記事の鵜呑み
5. **デザインループ** — 成果物の品質を根拠ベースで評価し、ループで保証する仕組み
6. **日常運用** — handoff、/eod、セッション管理の実際
7. **設定ファイル全文** — コピペして自分の環境に適用できる形で掲載

## この本で扱わないこと

- Claude Code自体のインストール方法や基本操作
- プロンプトエンジニアリングの一般論
- 他のAIツールとの比較

## 前提環境

- Claude Code v2.1系（2026年6月〜7月時点）
- macOS
- Obsidian（ナレッジベースとして併用しているが、必須ではない）

本書のコード・設定ファイルはすべて実際に動いているものからの抜粋です。
