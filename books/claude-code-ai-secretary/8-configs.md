## この章の使い方

この章には実際に動いている設定ファイルを掲載しています。コピペして自分の環境に適用し、必要に応じて内容を書き換えてください。

ファイルの配置場所は `~/.claude/` 配下です（Claude Codeのグローバル設定ディレクトリ）。

## CLAUDE.md（抜粋: 自律学習プロトコル部分）

CLAUDE.mdはClaude Codeが起動時に必ず読み込む常駐コンテキストです。全体は長いので、自律学習プロトコルの核心部分を抜粋します。

```markdown
## ★ 自律学習プロトコル(最重要)

このAI秘書は会話の中で勝手に成長する。
ユーザーが明示的なコマンドを打たなくても、以下を自動で実行する。

### A. 会話中の即時学習 (Real-time)

会話の流れの中で以下のいずれかを検出したら、
ユーザーに確認せず該当ファイルを Edit / Write ツールで直接更新する。

| 検出した情報 | 更新先 | 操作 |
|-------------|--------|------|
| 新しい社内用語/略語/コードネーム | glossary.md | 追記 |
| 新しい人物(名前・役割・関係) | people.md | 追記 |
| プロジェクトの状態変化 | Obsidian Projects/ | 更新 |
| 表明された好み・習慣 | preferences.md | 追記/上書き |
| 重要な意思決定とその理由 | decisions.md | 追記 |
| 学び・気づき・詰まったこと | lessons.md | 追記 |

### B. 自動更新のルール

1. 重複は避ける: 追記前に既存内容を読み、似た記述があれば統合
2. 確信ベース: 曖昧な情報は書かない。明確に表明されたもののみ
3. 報告は最小限: 重要な追加のみ1行報告。些末な追記は黙って実行
4. 1エントリずつ追記し、まとめて巨大な更新はしない
5. 削除は提案のみ: 自動で削除はしない
```

## メモリファイルのテンプレート

### glossary.md

```markdown
# 用語集 (Glossary)

> 社内用語・略語・コードネーム・専門用語の対訳と解説。
> 新しい用語が出たら追記する。

## フォーマット

### 用語
- 意味: 短い定義
- 文脈: いつ使う/誰が使う
- 関連: 他の用語へのリンク
```

### people.md

```markdown
# 人物録 (People)

> 仕事で関わる人々の最小限のメタ情報。
> プライバシー配慮: 個人情報(住所、電話、生年月日)は書かない。

## フォーマット

### 名前 (呼び方)
- 役割: 何をしている人か
- 関係: 自分とのつながり
- 文体: 接し方のメモ(敬語/タメ口、技術レベル等)
- 最終接触: YYYY-MM-DD
```

### decisions.md

```markdown
# 決定ログ (Decisions)

> 「なぜそう決めたか」を未来の自分が思い出すための記録。
> ADR (Architecture Decision Record) の個人版。

## フォーマット

## YYYY-MM-DD: 決定のタイトル
- 状況: 何が起きていたか
- 選択肢: 検討した案
- 決定: 何を選んだか
- 理由: なぜそれを選んだか
- 結果: (後から追記) 実際どうだったか
```

### lessons.md

```markdown
# 学び・失敗ログ (Lessons)

> 「同じ轍を踏まない」ためのログ。

## フォーマット

## YYYY-MM-DD: 短いタイトル
- 何が起きた:
- 何を学んだ:
- 次にどうする:
- 関連:
```

### handoff.md

```markdown
# 引き継ぎメモ (Handoff)

> 「昨日の自分から今日の自分へ」のメモ。
> このファイルは追記ではなく上書きで運用する。

## 最終更新: YYYY-MM-DD

## 直近の状況
（現在の方針・進行中タスク・最新の決定）

## 未完了タスク
（箇条書き）

## 朝一で確認すべきこと

## 注意・気を付けること
```

## エージェント定義の例

### research-loop.md

```yaml
---
name: research-loop
description: リサーチのループオーナー。
tools: Agent, Read, Write, Edit, WebSearch, WebFetch
model: claude-opus-4-6
maxTurns: 50
---
```

エージェント定義はYAMLフロントマター + Markdownの指示文で構成されます。`~/.claude/agents/` に配置します。

重要なパラメータ:
- **model**: 判断層にopus、実行層にsonnet
- **tools**: そのエージェントが使えるツールを制限する
- **maxTurns**: ループの暴走防止。research-loopは50、researcherは40

### researcher.md（指示の要点）

```markdown
与えられたトピックをWeb検索と既存ドキュメントの両方で調査し、
出典付きで構造化された要約を返す。

## 反復リサーチループ
ラウンド1: 広域検索（最低8クエリ、WebFetch必須）
  ↓ 自己評価
ラウンド2: ギャップ補完
  ↓ 自己評価
ラウンド3: 最終補完（必要な場合のみ）

## 停止条件
- 2ソース以上で回答
- 情報飽和
- 3ラウンド上限
```

2. 顧客セグメンテーション
3. 需要側pain/課題

## settings.json（メモリディレクトリの書き込み許可）

Claude Codeの `settings.local.json` でmemoryディレクトリへの書き込みを事前承認します。

```json
{
  "permissions": {
    "allow": [
      "Edit(~/.claude/memory/**)",
      "Write(~/.claude/memory/**)"
    ]
  }
}
```

これにより自律学習プロトコルが確認ダイアログなしで動きます。

## 最小構成で始める場合

全部を一度に導入する必要はありません。最小構成は以下です。

1. `~/.claude/CLAUDE.md` に自律学習プロトコル（セクションA + B）を書く
2. `~/.claude/memory/` に空のメモリファイル5つ（glossary, people, preferences, decisions, lessons）を置く
3. settings.jsonでmemoryディレクトリの書き込みを許可する
4. 普通に会話する

これだけで、会話するたびにメモリが蓄積されていきます。エージェント構造やリサーチループは、メモリが育ってから必要に応じて追加すればよいです。
